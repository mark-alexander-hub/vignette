2. HaveIBeenPwned Module

Security File: modules/hibp.php

Purpose

Checks if an email address has been exposed in known data breaches. Uses the HaveIBeenPwned API v3 maintained by security researcher Troy Hunt.

What It Detects

  • Which data breaches contained the email address
  • What types of data were exposed (passwords, addresses, phone numbers, IP addresses, etc.)
  • Whether the breach was "sensitive" (adult sites, health data, dating platforms)
  • Whether credentials were harvested by malware (stealer logs)
  • The date each breach occurred
  • Total number of accounts affected in each breach

4. Gravatar Module

Identity File: modules/gravatar.php

5. SerpAPI Module (Google Search)

Discovery File: modules/serpapi.php

Purpose

Searches Google for any public web pages that mention the target. Returns real Google search results as structured data via SerpAPI. This is one of the most powerful modules because it discovers information that no single API provides — news articles, forum posts, social media profiles, professional listings, and more.

What It Discovers

  • Public web pages mentioning the target (LinkedIn profiles, social media, news articles, blog posts)
  • Social media profiles that Google has indexed (often more accurate than username guessing)
  • Professional listings (company pages, directory entries)
  • Academic papers and publications
  • Forum posts and community contributions
  • Domain mentions and backlinks

Smart Social Extraction

The aggregator automatically extracts social media profiles from Google results using URL pattern matching. When Google returns a result like https://www.instagram.com/beastboy_m.i.n.a/, Vignette extracts the username and platform, marks it as Web Verified, and prioritizes it over username-guessed profiles.

8. DNS Module

Domain Intel File: modules/dns.php

Purpose

Fetches DNS records and analyzes email security configuration. Detects mail providers, hosting/CDN providers, and computes an email security score based on SPF, DKIM, and DMARC configuration.

What It Discovers

  • A, AAAA, MX, TXT, NS, SOA, CNAME records
  • Mail provider detection (Google Workspace, Microsoft 365, custom mail, etc.)
  • Hosting/CDN detection (Cloudflare, AWS, Azure, etc.)
  • SPF record analysis
  • DKIM detection
  • DMARC policy and enforcement level
  • Email security score (0-100)

9. SSL Module

Domain Intel File: modules/ssl.php

Purpose

Inspects the TLS/SSL certificate of a domain by opening a direct socket connection. Analyzes certificate authority, type, expiry, key strength, and subject alternative names.

What It Discovers

  • Certificate type: DV (Domain Validated), OV (Organization Validated), EV (Extended Validation)
  • Issuing Certificate Authority
  • Subject Common Name and Organization
  • Valid from/to dates
  • Key type (RSA or EC) and key size
  • Subject Alternative Names (SAN) — all domains covered by the cert
  • Certificate chain visualization

11. IPInfo Module

Identity File: modules/ipinfo.php

Purpose

Geolocates IP addresses and detects privacy services (VPN, proxy, Tor, hosting). For domain queries, the domain is first resolved to an IP via DNS.

What It Discovers

  • City, region, country, and GPS coordinates
  • ISP/organization name
  • AS number and name
  • Whether the IP is a VPN, proxy, Tor exit node, or hosting/datacenter IP
  • Timezone
Example Output — IP: 104.26.10.78

Location: San Francisco, California, US
Coordinates: 37.7749, -122.4194
ISP: Cloudflare, Inc.
AS: AS13335 (Cloudflare)
Privacy: Hosting/Datacenter IP

Trigger Conditions

Activated for IP and domain query types. For domains, the IP is resolved first via gethostbyname().

13. Gemini AI Module

AI File: modules/gemini.php

Purpose

Generates an AI-powered intelligence briefing from all gathered data using Google's Gemini (gemini-2.5-flash) model. The briefing reads like a professional intelligence analyst's summary.

What It Produces

  • Executive summary of the target's digital presence
  • Key findings and notable data points highlighted
  • Risk factors explained in plain language
  • Connections and patterns identified across data sources
  • Recommendations or areas of concern
Example AI Briefing — Email: developer@company.com

AI Intelligence Briefing

The target email is associated with a senior software developer based in Bengaluru, India. The individual maintains an active professional presence across multiple platforms.

Key Findings:

  • GitHub profile shows 42 public repositories with focus on PHP and JavaScript development
  • Active on 18 of 28 social platforms checked, with verified Instagram and LinkedIn profiles
  • Email was exposed in 2 data breaches (LinkedIn 2012, Adobe 2013) — both included passwords

Risk Assessment: Moderate (45/100). Primary concern is the password exposure in two major breaches. Recommend verifying that credentials have been updated.

15. Bulk Search

Purpose

Search up to 10 targets simultaneously. The full intelligence pipeline runs on each query.

How It Works

  1. Switch to "Bulk" mode on the search page
  2. Select a query type (all queries in the batch share the same type)
  3. Enter one query per line in the textarea (max 10)
  4. Click "Investigate All"
  5. View the summary card with aggregate stats and individual results
  6. Click "View Full" on any result to see the complete profile

Output

  • Total queries, success count, failure count
  • Average risk score across all results
  • Compact result cards: name/query, risk badge, source count, breach count
  • Each result links to the full profile view via the replay system
